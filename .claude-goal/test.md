# 验证清单

## Markdown 渲染

- [ ] **T1**: Claude 回复包含 `**加粗**` → 页面显示为粗体文本（非原始星号）
- [ ] **T2**: Claude 回复包含围栏代码块 ` ```js\nconsole.log('hi')\n``` ` → 显示为带语法高亮的代码块
- [ ] **T3**: 代码块左上角显示语言标签（如 "javascript"），右上角有复制按钮
- [ ] **T4**: 点击复制按钮 → 剪贴板获得代码内容，按钮短暂显示 ✓
- [ ] **T5**: Markdown 表格正确渲染为 HTML table（有边框/条纹）
- [ ] **T6**: 链接 `[text](url)` 渲染为可点击的 `<a>` 标签（target=_blank）
- [ ] **T7**: 内联代码 `` `code` `` 有 monospace 背景样式
- [ ] **T8**: 有序/无序列表正确缩进渲染
- [ ] **T9**: 引用块 `> quote` 显示带左边条样式

## 图片上传

- [ ] **T10**: 输入区域可见图片上传按钮（📎 或图片图标）
- [ ] **T11**: 点击按钮弹出文件选择器，选择 png/jpg → 输入区下方出现缩略图预览
- [ ] **T12**: 在输入框 Ctrl/Cmd+V 粘贴剪贴板图片 → 同样出现预览

- [ ] **T14**: 预览中有 × 按钮，点击可移除已上传图片

## 图片展示

- [ ] **T18**: 点击消息中的图片 → 弹出灯箱（overlay）显示原图，点击外部关闭

- [ ] **T24**: 正常的 Markdown 标签（strong、em、a、table 等）不被过度过滤

## 视觉一致性

- [ ] **T25**: 所有新增 UI 元素使用 CSS 变量（--bg, --fg, --border, --accent 等）

## 手动验证步骤

1. 启动服务 `node server.js`
2. 浏览器打开 http://localhost:4477
3. 新建一个会话，发送：`请用 markdown 格式回复，包含标题、代码块、表格和链接`
4. 观察回复渲染效果，逐一核验 T1-T9
5. 在输入区尝试粘贴截图、拖拽图片、点击上传按钮，核验 T10-T15
6. 发送含图片的 prompt 后检查网络面板确认数据格式
7. 手动在 chatPane 注入含 XSS payload 的假消息，核验 T22-T23
